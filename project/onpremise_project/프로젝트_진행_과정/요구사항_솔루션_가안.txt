가. 데이터 백업을 위한 서버를 구축하고 장애 발생시 원인 파악 및 대응을 위한 즉각적인 로그 백업
    - FTP 서버를 구축하여 데이터 백업 및 로그 백업

    - 감시중인 연결에 문제 발생시 EEM이 동작

        - 문제가 발생한 연결지점의 인터페이스 shutdown


나. 인터넷과의 통신 안정성 확보
    - 라우터에 ACL 정책 설정으로 외부 접근 제어
    - NAT 이용한 사설/공인 IP 주소 변환
    - HSRP 이중화 구성으로 장애 대비
        - BB2 와의 연결이 Active 상태이며 장애시 BB로 자동 전환


다. 동일한 구역의 네트워크 구성에서 각 부서 및 학과 워크 그룹 스위치에 대해 효율적인 이중화 구성
    - 이중화 구성
        - 이중화 구성으로 장애 발생시 즉각적인 우회 경로 확보로 고가용성 확보
        - Vlan을 이용하여 L2스위치와 L3스위치 연결 이중화
        - HSRP를 아용하여 L3 스위치와 백본 연결 이중화 (Active-Standby)
        - Link Aggregation 이용하여 L3 스위치간 연결 이중화


라. 구축된 시스템 호환 장비 도입을 통한 운영 안정성 강화 및 증설되는 미래기술관 네트워크 망에 신규 백본 스위치 도입
    - 2개의 백본 스위치, 7개의 멀티 레이어 스위치 도입
    - Core Layer, Distribution Layer, Access Layer 의 3-tier 네트워크 망 구성
    - OSPF 라우팅
        - Area 0 ~ 4 까지 총 4개의 구역으로 분리하여 관리 용이성 증대


마. 백본 스위치 사이에 멀티 라인 연결으로 기반 확대 및 고성능, 고가용성 운영 환경 마련
    - 본관과 미래기술관 사이 4개의 물리적 라인 연결
    - Link Aggregation
        - 2개의 라인을 1개의 논리적 라인으로 구성하여 대역폭 증대
        - 총 2개의 EtherChannel (4개의 물리적 라인 중 2개씩 Link Aggregation)
    - 2개의 EtherChannel HSRP 로 이중화 구성하여 안정성 확보


바. 신규 전산정보센터를 신설하여 각종 서버 서비스 제공
    - 서버 구축
        - 코드형 인프라 (Infrastructure as Code, IaC)
            - 코드 베이스의 서버 서비스 구축
            - 간단한 명령어 실행만으로 전체 서버를 10분여 시간으로 구축 가능
            - 각 서버의 설정을 파일형태로 분리하여 분석 및 관리 용이성 극대화
        - Docker container - 가상화 기술을 통하여 서버 서비스를 개별 구축
            - 효율성: 하드웨어 자원을 효율적으로 사용
            - 독립성: 서비스간 충돌 최소화 및 호스트 하드웨어의 안정성 확보
            - 확장성: 향우 필요에따라 스케일 아웃이나 오케스트레이션이 가능
            - 안정성: 컨테이너간 개별 볼륨 사용으로 데이터 손실 위험 없이 손쉽게 재구축 가능
        - Docker-compose
            - 연동하여 동작하는 컨테이너들을 하나의 코드로 구축 및 관리
            - 웹서비스(Web-server, AP, DB)의 3개의 컨테이너가 연동
    - 서버 서비스
        - FTP: 데이터 백업 및 로그 백업
        - Web-sevice: MZC 대학의 웹 서비스
        - DHCP: VLAN 기준으로 엔드 디바이스의 IP 주소를 자동으로 할당
