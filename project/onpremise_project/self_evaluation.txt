1. 주요 담당 업무 - 서버 구축
    - DHCP: 네트워크 구성 정보를 각 구역별로 분리하여 자동 할당
    - FTP 서버 - 네트워크를 통해 파일을 저장할 수 있는 환경 구성
    - Web Service: docker compose 를 이용하여 Nginx, Django app, Postgres DB의 3개의
    컨테이너를 연동하여 구축 웹 서비스 구축. (Test Sample - index.html, 어드민 계정, 2개의 일반 유저계정)
    - SLA / EEM / FTP 를 이용하여 장애발생시 즉각적으로 장애 상황을 파악할 수 있는 내용을 FTP 서버로 전송하는 로직 구현
    - 작성한 서버를 다른 팀원이 쉽게 적용할 수 있도록 모든 과정을 코드화 하여 공유(Dockerfile, docker-compose
    .yml, 환경 구성 및 컨테이너 생성/삭제 Shell script)



2. 요구사항 솔루션 정리
    - 서버구축이 주요 업무였기 때문에 네트워크 망 구축 진행사항과 사용된 기술 이해가 등한시 되지 않도록 노력
    - 각 기술이 어디에 어떻게 어떠한 목적으로 사용되었는지 총 정리하는 문서의 필요성을 제안하고 작성(-> 기술보고서_요구사항_솔루션
    .docx)
    - 팀원에게 받은 요구사항과 요구사항 솔루션 리스트의 초안을 바탕으로 망 구성에 사용된 기술을 파악하여 각 솔루션 항목에 분배 정리
    - 팀원과의 커뮤니케이션을 통해 정리한 내용의 오류를 수정, 내용 보완, 문장 다듬기 과정을 통해 완성



3. 팀원 서포트(두희님)
    - 프로젝트에 소외되지 않고 적극적인 참여가 가능하도록 서포트
    - Draw.io 툴의 사용법 설명하였고, 이를 통해 팀원들의 구상을 바탕으로 다이어그램 이미지(초안 및 1차 수정안)를 작성할 수
    있도록 도움
    - 두희님이 포트맵 작성 과정에서 겪은 어려움을 극복하기 위해 함께 팀원에게 조언을 구하며 전체 망 구성에 대해 학습하고 토의
