network 종류를 두가지로 나눈다
    
    
    1. 서비스를 형태 기준으로 나논다.
        - LAN:
            - Local Area Network
            - 지금 교육 과정에서 Network 는 LAN으로 가정한다
            - LAN, Network, Broadcast Domain
        - WAN: Wide Area Network
    
    
    
    2. 서비스를 제공하는 유형으로 나눈다. 
        
        유형 분리:
            - peer to peer 
                - 대표적인 케이스:
                    - 토렌트: 서버와 클라이언트의 구분이 없다
            - server and client
        
        참고: 개발자가 고민하는 영역








IPv4 Address





* 1.1.1.0/24
    -> 24번쨰 비트 까지가 네트워크 주소이다.
    -> / 가 붙은 순간 클래스를 따지지 않는다.
    -> 마지막 0: 네트워크를 식별하고 있다
    -> 가용ip: 254개
    -> 브로드캐스트: 1개



* MAC address
    -> 48bit



* 인터넷이 안될때 프로토콜
    - 장치 연결상태 확인 (랜선, 와이파이)
    - 장치의 정상동작여부 확안
        - icmp ==> ping
        - ping 127.0.0.1
        - ping localhost
        - ping gateway
        - ping 168.126.63.1
        - Route check
            - tracert 168.126.63.1 (Windows)
            - traceroute 168.126.63.1 (macOS)










* TCP/IP 계층
4 layer  - Port     - L4 switch (L4 switch)
3 layer  - IP       - L3 switch (Router)
2 layer  - MAC      - L2 switch (switch)
1 layer  - Physical - L1 switch (hub)



*
    3 layer - Collision Domain(o) Broadcast Domain(o)
    2 layer - Collision Domain(o) Broadcast Domain(x)




* OSI 7 layer
7 layer - Application
6 layer - Presentation
5 layer - Session
4 layer - Transport
3 layer - Network
2 layer - Data Link
1 layer - Physical
