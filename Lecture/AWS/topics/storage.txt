Amazon S3

    - Amazon Simple Storage Service


S3 bucket

    - 거의 완벽한 내구성을 보장하므로 HA 고려없이 정적인 웹 페이지 구성 가능
    - 람다와 연동할 경우 동적 웹 서비스도 가능



버킷 이름은 글로벌 네임스페이스에서 유니크해야 함
    - Bucket Name 을 바탕으로 객체 접근 URL 생성하기 때문

버킷의 위치는 리전에 종속




리소스 기반 정책
    - IAM, S3, AWS Organizations 서비스 제어 정책 기반 엑세스 제어
    - JSON 기반 액세스 정책



S3 액세스 포인트

    - S3 설정 -> 속성


S3 비용 청구 방식

    - 비용 관련 요인
        - 저장 용량
        - API 요청 회수
        - 데이터 전송 (인바운드 무료, 아웃바운드 유료)

    - 저장 용량/API 요청 비율에 따라 플랜을 나누어 놓음






* One Zone IA
    - 가용 영역이 1개
    - 안정성이 떨어진다, (다른 제품은 3개의 가용영역이 3개로 나누어 저장된다)




* S3 버전관리기능

    - 속성에서 버전 관리 활성화 체크
